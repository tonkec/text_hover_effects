About

Text Hover Effect is a Sass mixin library that consists of effects for typography that are triggered on hover. Effects are made with pseudoelements ::after and ::before and CSS3 transitions. See all of them here.

Getting Started

Download or clone repo from Github and include effect.scss file. In order effects to work you need to have partials directory in your project directory on the same level as effect.scss file. Effects.scss imports all of the mixins. If you need only several effects, delete unnecessary imports. All effects must have _base.scss imported because it has transition-opt and line-opt mixins mandatory for all effects.

           
             
              /* style.scss */

              @import "effects";
            
          
        
         
           
            /* structure */

            |--partials/
            |--effects.scss
            |--style.scss

How To Use

1. The Sass Way

After you have all the files downloaded, you are ready to use effects. Include transition-opt to an element you wish to apply effect for. Then add effect. For example:

       
         
           .element { 
           @include transition-opt;
           @include underline-right;
         }
Result

Be careful with using CSS #id selector because it will overwrite several styles. It is recommended to use classes instead.

2. The CSS Way

CSS styles are also available. After you cloned/downloaded repo from Github, you can use them as well. All the CSS files are included in CSS/effects directory. Choose one you want and copy and paste the code into your project file. Add class to your element and you are good to go!

     
       
         .underline-left:after { 
         content: "";
         position: absolute;
         z-index: -1;
         width: 3px;
       }

       .underline-left:after {
       right: 0;
       bottom: 0;
     }

     .underline-left:hover:after {
     width: 100%;
     height: 2%;
   }
Result

Customize

Mixin transition-opt has several options. You can change each of them. These are defaults

   
     
       .element { 
       @include transition-opt(
       $font-size: 4em ,
       $font-weight: 700,
       $color: #2C3E50,
       $hover-color: #FFF,
       $border-after: 0%,
       $border-before: 0%,
       $display: inline,
       $transition-color: #FFF,
       $transition-speed: 1s,
       $tranition-easing: ease-in-out,
       $delay-after: 0s,
       $delay-before: 0s,
       $padding: 10px
       )};

Change whichever property you want but be careful with changing display property. It is recommended to use display: inline because then width of transition will be equal to the width of an element. If you change to display: block, it will change the width of transition and then you must change width and margin of an element. For example:

   
     
       .element { 
       width: 100%;
       @include transition-opt(
       $display: block
       )

       @include underline-right;
     };
Result 
   
     
       .element { 
       width: 50%;
       margin: 0 auto;
       @include transition-opt(
       $display: block
       )

       @include underline-right;
     };
Result

    Under the hood

    All transitions are made with pseudoelements ::before and ::after. Each element that has transition-opt mixin included will have those two pseudoelements. Mixin transition-opt sets the base for transition. It allows you to set the options. It triggers hover and focus event. You can see code in _base.scss

    Each transition includes line-opt mixin which sets width and height of pseudoelements. Default width is 3px, and default height is 2%. Line-opt depends on type of transition. Some transitions have only one pseudoelement. Line-opt mixin allows to choose which pseudoelement should be created for particular transition. For example, overline-left transiton is created with ::after

       
     
      @mixin overline-left {
      @include line-opt(
      pseudoafter,
      $transition-height: $height
      );

      &:after {
      right: 0%;
      top: 0%;
    } 

  }
    Result

    First we define mixin overline-left. Then we include line-opt mixin and define type of pseudoelement which is here pseudoafter. After that, we define transition width. Since default transition-width in line-opt is 100% we do not need to set it explicitly. But default transition-height in line-opt is also 100% and we do not want that. We want it to remain the same. That is why we set it to height of an element which is 2%.

    Some transitions needs to have both pseudoelements such asoverline-closing. Pseudoelements are positioned differently.

       
     
      @mixin overline-closing {
      @include line-opt(
      pseudoafter,
      $transition-height: $height,
      $transition-width: $half-width
      );

      @include line-opt(
      pseudobefore,
      $transition-height: $height,
      $transition-width: $half-width
      );

      &:after {
      left: 0%;
      top: 0%;
    }

    &:before {
    right: 0%;
    top: 0%;
  } 

}
    Result

    Line-opt also sets the width and height of pseudoelement on hover and focus, that is, width and height of transition. Since default display property is set to inline, transition width or height will never be wider or higher than element's width or height. That is why transitions vary in maximum height or width. In other words, when hovered or focused, some pseudoelements have only maximum height, maximum width and some have both. For example, underline-right transition has maximum width set to 100%, while height is set to default, that is 2%. Some have maximum height set to 100%, while width is set to default, that is 3px.
